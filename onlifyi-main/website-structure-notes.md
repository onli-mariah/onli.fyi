# Website Structure Analysis

## Current Website Format

The existing Onli use case website has the following structure:

### Homepage Structure
1. **Hero Section**
   - Large headline: "the revolution you never saw coming."
   - Subtitle explaining Onli's actual-possession technology
   
2. **The Big Idea Section**
   - Explains Onli's fundamental concept
   - Emphasizes actual possession vs ledgers/blockchain
   
3. **Why Does It Matter Section**
   - Explains the value proposition
   - Introduces the use cases

4. **Use Cases Section**
   - Numbered use cases (01, 02, 03, 04, 05)
   - Each use case has:
     - Number prefix (01, 02, etc.)
     - Question-format headline
     - Detailed paragraph explaining the scenario
     - Specific metrics and results
     - Links to "Read whitepaper" and "Calculate savings"

### Use Case Format Pattern

Each use case follows this structure:
```
[Number]

### [Question headline]

[Detailed paragraph with:
- Company/scenario description
- Problem statement
- Onli solution
- Specific metrics and results
- Value proposition]

[Links to resources]
```

### Current Use Cases:

## Format for New Marketplace Comparison Use Case

The marketplace comparison should be added as use case #06 following the same pattern:

**Number:** 06

**Question Headline:** "What if you could build a marketplace for 99% less?"

**Content Structure:**
- Opening scenario (organizations evaluating blockchain platforms)
- Problem statement (hidden costs, development requirements)
- Onli solution (all-inclusive platform)
- Specific metrics (TCO comparisons across scales)
- Value proposition (95-99% savings)

**Links:**
- Read whitepaper → link to full report
- Calculate savings → (placeholder or calculator)
